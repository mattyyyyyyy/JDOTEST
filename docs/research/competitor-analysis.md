## TL;DR · 一句话结论

> **车机内嵌购物没有现成的"行业标准"可抄。**
> 真正成熟的是「车机系统 HMI 规范」（AAOS / 鸿蒙 / NHTSA），它们规定了"驾驶员能看什么、点什么、停留多久"。而**「车内电商」这个垂直品类在 2025–2026 仍处于早期**：海外几乎不存在，国内只有蔚来 / 小鹏 / 比亚迪等做了车机商城但内容多为车主周边而非通用电商。这给了我们 Demo 的机会，**也意味着设计规范要以 HMI 安全规范为主、电商交互模式为辅**。

## 关键 Design Pattern 命中率（基于本次调研覆盖的样本）

| Pattern | 命中率 | 备注 |
|---|---|---|
| 深色默认 | 7/7 | 行业一致 |
| 卡片化首页 | 6/7 | 仅特斯拉走 list/dock 风格 |
| 横屏分栏（左导航 + 主内容） | 5/7 | 主流模式 |

---

## 我们应该效仿的 3 件事

1. **卡片化首页 + 三大常驻区**（小米 SU7 模式）：车机首页天然就是「巨型组件画布」，不是手机的"无限下拉"。三大区是 `常买/再买一次`、`分类入口`、`促销/秒杀`，**位置固定 + 可隐藏**。
2. **行车态强降级**（鸿蒙 / NHTSA 模式）：单步操作 ≤ 2s 完成、全流程 ≤ 12s。下单走"默认地址 + 默认免密 + 历史 SKU"三件套。
3. **横屏分栏（左 sticky 导航 + 主内容）**：5/7 命中。比上下滚效率高得多，符合横屏视觉扫描节奏。

## 我们应该规避的 3 件事

1. **不要做"手机商城的横屏放大版"**：淘宝/京东那种密集瀑布 + 无限下拉，对车机来说是反模式（视觉扫描成本高、单次操作时间长）。**信息密度反而要降低**。
2. **不要依赖手势 / 长按 / 双击**：颠簸路况下不可靠（AAOS 显式建议），全平台禁手势。
3. **不要做行车时的视频/动效/弹窗**：法规明令禁止、用户体感差。

## 各竞品详解

### 1. NIO 蔚来商城 / NIO App

- 资料来源：
  - [蔚来车商城（M 端）](https://m-car-mall.nio.com/)
  - [蔚来全新 ES8 智能系统介绍](https://www.nio.cn/es8)
  - [NIO Day 2025 第三代 ES8](https://news.qq.com/rain/a/20250922A02Z8500)
- **车机系统**：NT.Cedar（雪松），第三代数字架构 + 整车 OS，搭载于 2025 年发布的全新 ES8
- **设计语言**：纯净、温暖、未来感、精致；第三代 "Design for Success" 设计理念
- **商城形态**：独立站 `m-car-mall.nio.com`（车手互通，车上手机均可访问），偏车主周边 + NIO Life 自营商品
- **典型布局**：上方主视觉 banner → 分类入口 → 热销 → 商品 grid，深色背景，按钮主色为蔚来灰蓝
- **可借鉴**：
  1. 商城与车机系统视觉语言强一致（不像"塞进去的网页"）
  2. 商品图大留白多，符合车内"瞥一眼能识别"的需求
- **应规避**：
  1. 当前商城仍是手机优先，横屏适配较弱
  2. 商品库存以 NIO Life 自营为主，泛品类不丰富

### 2. 理想汽车 车机

- 资料来源：
  - [车载 HMI 设计干货！理想汽车座舱界面再设计](https://www.uisdc.com/lixiang-hmi-design)
  - [理想 ONE 四屏交互保姆级评测](https://zhuanlan.zhihu.com/p/170606625)
  - [42how：理想 ONE 四屏交互 21 项升级](https://www.42how.com/en/article/2898)
- **车机系统**：基于 Android 深度定制，**四屏交互**（中控 + 副驾 + 仪表 + HUD）
- **应用生态**：第三方 APP 约 10 个，**官方暂不支持应用商店让用户自由扩展**，每个应用都要做深度优化（UI 风格 + 语音识别）
- **横屏特性**：进入全屏应用后可点右下角进入分屏模式，主副窗口大小可拖动
- **设计原则**（评测总结）：对比聚焦 / 重复一致 / 对齐有序 / 邻近关联
- **关键启示**：理想的"封闭应用生态 + 每个应用都要 OEM 严格审查"恰好给我们 Demo 的定位提了一个反向参考——**我们做的是"通用电商"，必须比理想更克制、更安全，才有机会被这种严格车厂接纳**
- **可借鉴**：四屏分工思路（副驾屏可以做完整购物，主驾屏强降级）— **写入 PRD v0.5 的远景**
- **应规避**：四屏开发成本极高，Demo 阶段不做

### 3. AITO 问界 / HarmonyOS 智能座舱

- 资料来源：
  - [华为乾崑 鸿蒙座舱](https://auto.huawei.com/cn/harmonyspace)
  - [问界 M9 官网](https://aito.auto/model/m9/)
  - [鸿蒙智能座舱设计规范要点](https://ai6s.net/69a687380a2f6a37c594c652.html)
  - [HarmonyOS 智能座舱体验是怎样炼成的](https://www.cnblogs.com/HarmonyOSDev/p/17043213.html)
- **市场地位**：2025 年累计搭载 300 万辆+（问界 / 阿维塔 / 极狐等多品牌）
- **设计规范（关键定量）**：
  - **交互复杂度**：单次操作 ≤ 2s，整个流程累计 ≤ 12s
  - **安全区**：基于驾驶员视觉行为划分主/次安全区（头部左右上下 15° / 30°）
  - **信息层级**：必须有字号差、必须有低/高层级控件的颜色对比度
  - **沉浸式开发**：状态栏 + 底部导航区 = "避让区"，其余为"安全区"，UI 元素必须避让，颜色风格保持一致
- **应用接入**：ArkUI 开发，需遵循「交互/显示/控制/数据」四方面规范，地图导航类强制语音控车适配
- **可借鉴**：
  1. **2s/12s 规则直接成为我们行车态全流程预算约束**
  2. **主/次安全区**概念可用于"驾驶员位（左半屏）放降级版、副驾位（右半屏）放完整版"（未来阶段）
  3. 沉浸式 / 避让区设计写进 IVI Shell 模块
- **应规避**：鸿蒙是 ArkTS 封闭技术栈，与我们 React + H5 路线不重合，**不直接 copy 实现，只学规范**

### 4. Tesla 车机

- 资料来源：
  - [Tesla 新 UI 首发体验 - Electrek](https://electrek.co/2021/06/11/tesla-new-user-interface-first-look/)
  - [Tesla Entertainment System - Shop4Tesla](https://www.shop4tesla.com/en/pages/tesla-entertainment-system)
  - [A Deep Dive into Tesla's UI - Medium](https://medium.com/@ethanwwm/a-deep-dive-into-teslas-user-interface-9c4aa3e6a4ab)
- **关键发现**：**特斯拉车机没有"购物商城"**，娱乐系统仅含 Arcade（游戏）/ Theater / 卡拉 OK，购物属于刻意不做的品类
- **设计语言**：极简主义、巨大触屏（横屏 15"）、按钮极少、依赖滑动 + 卡片切换
- **可借鉴**：
  1. 视觉极简 — 信息密度低、留白多
  2. 行车时只允许娱乐系统在停车后用（"Park to use"）— **直接对应我们「行车态绝大多数功能锁死」**
- **应规避**：
  1. 全面依赖触屏滑动 — 不符合 AAOS 反手势建议；我们走可见按钮 + 一次点击
  2. Tesla 的 UI 频繁大改（用户怨声载道），**我们要选稳定布局，不追潮流**

### 5. 小鹏 Xmart OS 5.0（天玑）

- 资料来源：
  - [小鹏 X9 XOS 5.0 HMI 界面分析（知乎）](https://zhuanlan.zhihu.com/p/677551751)
  - [对话小鹏汽车刘毅林：详解 Xmart OS 研发](https://chedongxi.com/p/238987.html)
  - [小鹏应用商店新增雷石 KTV / 全民 K 歌（42how）](https://www.42how.com/en/v2post/3760)
  - [小鹏 Xmart OS 百科](https://www.yiche.com/baike/1472.html)
- **车机系统**：基于 Android，**开放应用市场**（与理想路线相反）
- **应用生态**：游戏 / KTV / 音乐 / 有声书 / 支付宝小程序适配
- **桌面布局**：XOS 5.0 提供新桌面（3D 车模 + 地图 + 分屏卡片）
- **可借鉴**：
  1. **支付宝小程序适配模型**对我们极有参考价值——证明"小程序级 H5"可以在车机内顺畅跑，这就是我们的形态
  2. 横屏 P7 模式 vs 竖屏 P5 模式的差异化（说明横屏需要专门设计，不是把竖屏拉宽）
- **应规避**：早期 Xmart OS 字号偏小、按钮密集，被多方评测吐槽——我们字号要大、按钮要疏

### 6. 小米 SU7 澎湃 OS

- 资料来源：
  - [小米 SU7 官网](https://www.xiaomiev.com/su7)
  - [小米澎湃智能座舱介绍](https://www.xiaomiev.com/smartcabin)
  - [小米 SU7 智能座舱创新解析（腾讯）](https://news.qq.com/rain/a/20240711A06K1N00)
  - [小米 SU7 车机生态壁垒（新出行）](https://m.xchuxing.com/article/111489)
- **车机系统**：澎湃 OS，手机/车机/智能家居互通
- **设计精华（卡片化首页）**：
  - 首页 3 个常驻卡片：**导航 / 控车 / 多媒体**
  - 卡片**支持排列顺序自定义、可拖动调整大小、单卡片可全屏显示**
  - "多任务柔性框架"：应用可随意拖拽、扩展多桌面、调整窗口大小
- **性能指标**：拉开车门→中控屏点亮 **1.49 秒**
- **可借鉴**（最强参考）：
  1. **三常驻卡片模型** → 直接对应我们「常买/分类/秒杀」三大区
  2. **卡片可大可小**的弹性是我们行车/停车切换的天然实现路径（卡片放大 = 行车态简化）
  3. 启动速度 1.49s = 我们 FCP ≤ 1.5s 目标的现实参照
- **应规避**：澎湃 OS 强依赖小米生态账号体系，我们不能照搬

- 资料来源：
  - [Polestar 2 Technology - Polestar 官网](https://www.polestar.com/global/polestar-2/technology/)

## 行业 HMI 规范（决定我们硬约束）

### Android Automotive OS (AAOS)
- 来源：[AAOS Sizing - Google for Developers](https://developers.google.com/cars/design/android-auto/design-system/sizing)
- 来源：[Driver Distraction Guidelines - AOSP](https://source.android.com/docs/automotive/driver_distraction/guidelines)
- 来源：[Snapp Automotive: How big should touch areas be](https://www.snappautomotive.io/blog/how-big-should-touch-areas-in-car-interfaces-be)

| 规范 | 数值 | 说明 |
|---|---|---|
| 最小触控目标 | **76 × 76 dp** | 可在高度方向略大、宽度略小（适应上下颠簸） |
| 触控目标间距 | ≥ 23 dp | 避免误触 |
| 图标尺寸 | 主 44 / 副 36 / 三级 24 dp | 三级语义 |
| 元素高度同层 | 触控目标内所有元素必须同一 elevation | 防止视觉错位 |

### NHTSA Driver Distraction (Phase 1)
- 来源：[NHTSA Visual-Manual Guidelines (Federal Register 2013)](https://www.federalregister.gov/documents/2013/04/26/2013-09883/visual-manual-nhtsa-driver-distraction-guidelines-for-in-vehicle-electronic-devices)
- **2/12 规则**：单次视线离开道路 ≤ 2 秒，全任务累计 ≤ 12 秒
- **禁止性任务清单**：显式列出"驾驶时不应允许"的任务（如视频播放、长文本输入、地图自由滚动）

### HarmonyOS 智能座舱
- 来源：[HarmonyOS Cockpit 设计要点](https://ai6s.net/69a687380a2f6a37c594c652.html)
- 单次操作 ≤ 2s，累计 ≤ 12s（与 NHTSA 一致，可见这是行业事实标准）
- **主安全区**：驾驶员头部左右/上下 15° 内
- **次安全区**：驾驶员头部左右/上下 30° 内
- **避让区 vs 安全区**：状态栏/导航栏 = 避让区，UI 必须避开关键内容

## 给我们 Demo 的设计建议（最核心部分）

### 1. 色彩体系
- **深色默认**（与 7/7 竞品一致）
- 背景：`#0B0F14`（接近纯黑但保留蓝调，长时间观看不刺眼），灵感来自 Tesla / 小米 SU7
- 卡片表面：`#161B22`（与背景 2 阶差，符合鸿蒙信息层级规范）
- 主色 / 品牌强调：**电光青 `#00D4FF`**（车机常见的科技蓝青，便于在白底/黑底都高对比；蔚来用蓝灰、小米用青蓝，我们靠中间）
- 辅色：警示橙 `#FF6B35`（行车态警示用）、成功绿 `#22C55E`、错误红 `#EF4444`
- **对比度强制 ≥ 7:1**（WCAG AAA），文字白色 `#F5F7FA` 在主背景 `#0B0F14` 上对比度 17:1

### 2. 字号体系（基础 18px，比手机重一档）
| 角色 | 字号 | 行高 |
|---|---|---|
| 价格 / 主标题 | **40px** | 1.2 |
| 二级标题 | **32px** | 1.3 |
| 卡片标题 | **24px** | 1.4 |
| 正文 / 列表 | **20px** | 1.5 |
| 辅助文字 | **18px**（绝对下限） | 1.5 |
| 行车态正文 | **24px** 起步（比停车态升一档） | 1.5 |

> 依据：AAOS 16dp 基础字号在 200ppi 车机大屏上对应约 18-20px，鸿蒙规范要求层级有明显字号差。

### 3. 横屏栅格（针对 1920×720 基准）
- 12 列栅格，gutter 24px，外边距 40px
- **左导航**：宽 240px（固定 sticky），5-7 个一级入口
- **主内容**：剩余 12 列减导航宽
- **首页卡片**：分 6 列、4 列、3 列三档（小米三常驻 + 我们的「再买一次/分类/秒杀」三大区）
- 商品卡片宽高比 **4:5**（图 + 标题 + 价格 + 按钮，符合车内一瞥识别）

### 4. 组件库
- **按钮**：最小 88×88px（超 AAOS 76dp，留余量）；主按钮高度 88px、次按钮 72px、纯文本按钮 64px
- **卡片**：圆角 16px，阴影禁用（深色背景下阴影无意义，改用 1px 边框 `#21262D`）
- **导航 Tab**：竖向 sticky，单 tab 高度 88px，icon 44dp + 文字 20px
- **表单**：行车态禁用整个表单；停车态输入框高度 80px，错误提示永久显示在下方而非 tooltip

### 5. 行车态降级（综合鸿蒙 + NHTSA + Tesla）
- **单步 ≤ 2s 完成**，全流程 ≤ 12s
- **路径砍到三步**：「再买一次」→「确认（默认地址 + 默认免密）」→「下单成功」
- **隐藏**：banner / 视频 / 闪烁动效 / 输入键盘 / 复杂表单 / 评价 / 详情 Tab
- **保留**：首页常买卡片、订单状态、扫码登录
- **字号 +1 档、按钮 +1 档**（24→28、88→96 高度）
- **配色不改**（避免视觉刺激切换）

### 6. 横屏布局与移动电商的 3 个关键差异
| 维度 | 手机商城 | 我们（车机横屏） |
|---|---|---|
| 主滚动方向 | 垂直无限下拉 | **水平横向滑（卡片组）+ 垂直分页**（避免无限滚带来的"看不到底"焦虑） |
| 信息密度 | 高（一屏多 SKU） | **低**（一屏 6-8 SKU，留白 ≥ 24px） |
| 主操作位 | 底部一长条「立即购买」 | **右下浮按**（右手大拇指自然落点；中控屏副驾右手最近） |
