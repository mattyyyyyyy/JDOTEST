# 车机电商 · 交互模式 / 状态矩阵 / 决策树 Interaction Patterns v1.0

> 日期：2026-05-26 · 状态：Draft
> 上游：[feature-spec.md](../feature-spec.md)（功能+API 权威源）· [page-spec.md](./page-spec.md)（视觉布局）· [design-system.md](./design-system.md)（token+组件）
> 下游：`apps/h5/src/modules/*` · `packages/ui-components/*`
>
> **本文件目的**：补 feature-spec（按页面拆功能）和 design-system（按组件拆 token）之间的**中间层**——
> 1. **跨页面交互模式**：同一行为在多页出现时，规则必须统一
> 2. **多维状态矩阵**：页面 × (行车/停车) × (在线/离线) × (已登/未登) 的完整行为表
> 3. **决策树**：开发者拿到需求后不需要猜，按树走就能确定路径
>
> 开发任何页面/组件时，三步走：feature-spec 看"做什么" → **本文件**看"什么条件走什么路" → design-system 看"长什么样"

## 一、跨页面交互模式 Cross-Page Patterns

### IP-01 再买一次 Quick Rebuy

**出现页面**：P-01 首页 B-105、P-04 商品详情（行车态）、P-08 订单列表、P-09 订单详情、P-13 行车态首页

**统一规则**：

| 维度 | 行为 |
|---|---|
| 数据源 | `GET /api/v1/me/frequent-items?limit=N`；订单页/详情页的"再买一次"走 `POST /api/v1/cart/quick-add { skuId, qty: 1 }` |
| SKU 默认值 | 取用户**上次购买的 SKU**（`/me/last-purchased-sku/:pid`），若无历史则取商品默认 SKU |
| 数量 | 固定 1，行车态不可改；停车态可在购物车改 |
| 地址/支付 | 行车态：锁定默认；停车态：走正常结算流 |
| 已下架 | 该项灰显 + 「已下架」标记，不可点击 |
| 库存不足 | Toast「库存不足，已加入购物车待确认」→ 跳购物车 |
| 未登录 | 弹登录弹层，登录后回到当前页继续 |

**各页面差异**：

| 页面 | 形态 | 后续路径 |
|---|---|---|
| P-01 B-105 | 横向滑动卡片，按钮文案"再买一次" | 加购 → Toast → 停留本页 |
| P-04 行车态 | 单个大按钮替换原双 CTA，文案"一键购买" | `POST /api/v1/orders/quick { skuId }` → 默认地址+免密 → 支付结果页 |
| P-08 订单列表 | 订单卡片内"再买一次"文字按钮 | 加购 → Toast → 停留本页 |
| P-09 订单详情 | 底部操作栏"再买一次"按钮 | 加购 → Toast → 停留本页 |
| P-13 行车态首页 | 3 张巨卡 + 120px"一键购买"按钮 | `POST /api/v1/orders/quick { skuId }` → 默认地址+免密 → 支付结果页 |

### IP-02 登录拦截 Auth Gate

**触发场景**：任何需要用户身份的操作（加购、下单、查看订单、个人中心）

**统一规则**：

```
用户触发需登录操作
  ├─ 已登录 → 正常执行
  └─ 未登录 → 弹登录弹层（/login 作为 modal）
       ├─ 扫码成功 → 关闭弹层 → 执行原操作
       ├─ 手机号登录成功 → 同上（仅停车态可用）
       └─ 取消登录 → 回到原页面，不执行操作
```

**各页面差异**：

| 页面 | 触发点 | 弹层形态 |
|---|---|---|
| P-01 首页 | 点"再买一次"、点头像 | 右侧 Drawer（宽 600px） |
| P-04 商品详情 | 点"加购"/"立即购买" | 右侧 Drawer |
| P-05 购物车 | 点"去结算" | 右侧 Drawer |
| P-07 支付 | 进入页面即检查 | 全屏覆盖（支付必须登录） |
| P-08/P-09 订单 | 进入页面即检查 | 全屏覆盖 |
| P-10 个人中心 | 进入页面即检查 | 全屏覆盖 |
| P-13 行车态首页 | 自动在卡片区显示二维码（替代登录弹层） | 内嵌二维码，无弹层 |

**登录后状态恢复**：

- 弹层关闭后，自动重试被拦截的操作（如重新 `POST /api/v1/cart/items`）
- 购物车本地数据在登录后自动合并（CartStore 深模块处理）
- URL 不变，不跳转

### IP-03 地址选择 Address Selection

**出现页面**：P-06 结算页、P-04 商品详情（自提点）

**统一规则**：

| 条件 | 行为 |
|---|---|
| 有默认地址 | 自动选中默认地址 |
| 无默认地址但有地址 | 自动选第一个 |
| 无任何地址 | 引导"新增地址" |
| 行车态 | 锁定默认地址，不允许切换 |
| 停车态 | 允许在地址簿中切换 |
| 选中地址变更 | 重新计算运费 + ETA |

**"送到车上"特殊逻辑**：

- 仅当 JS Bridge 返回了车辆位置时可用
- 地址自动填充为「车辆当前位置」，不可手动编辑坐标
- 行车态下优先推荐"送到车上"（如果 Bridge 可用）

### IP-04 支付方式选择 Payment Method

**出现页面**：P-06 结算页（选择）、P-07 支付页（执行）

**统一规则**：

| 条件 | 默认支付方式 | 可切换 |
|---|---|---|
| 已绑定免密 | 免密一键支付 | 停车态可切换为扫码 |
| 未绑定免密 | 扫码支付 | — |
| 行车态 | 仅免密（未绑定则提示"请停车后完成支付设置"） | 不可切换 |

**支付路径决策**：

```
提交订单成功 → 进入 P-07 支付页
  ├─ 免密支付
  │    ├─ 有免密 → 显示大按钮「点击支付 ¥XXX」→ POST confirm → 结果
  │    └─ 无免密 + 行车态 → 显示"请停车后完成支付设置" → 停车后自动刷新
  └─ 扫码支付
       ├─ 停车态 → 显示二维码 + 倒计时 5min
       └─ 行车态 → 不走此路径（由上面的决策树保证不会到这）
```

### IP-05 购物车同步 Cart Sync

**出现场景**：加购、改数量、删商品、登录前后、断网恢复

**统一规则**：

```
本地操作（增/删/改）
  ├─ 在线 → 即时同步到服务端
  │    ├─ 成功 → 本地标记 confirmed
  │    └─ 失败 → 本地标记 pending + 后台重试（最多 3 次，指数退避）
  └─ 离线 → 写入 IndexedDB + 标记 pending
       └─ 网络恢复 → 自动批量同步 pending 操作

登录后合并
  ├─ 本地有 pending items → 合并到服务端购物车
  │    ├─ 同 SKU → 数量取较大值（不叠加）
  │    └─ 不同 SKU → 直接追加
  └─ 合并冲突 → 以服务端数据为准，本地标记为 resolved
```

**各页面行为**：

| 页面 | 操作 | 同步时机 |
|---|---|---|
| P-01 首页"再买一次" | quick-add | 即时 |
| P-04 商品详情"加购" | add-item | 即时 |
| P-05 购物车改数量 | patch-item | debounce 300ms 合批 |
| P-05 购物车勾选 | patch-item selected | debounce 300ms |
| P-05 购物车删除 | delete-item | 即时 |

### IP-06 搜索 Search

**出现页面**：P-03 搜索页、M-01 顶栏搜索框

**统一规则**：

| 条件 | 输入方式 | 联想 | 历史记录 |
|---|---|---|---|
| 停车态 + 在线 | 键盘 + 语音 | 200ms debounce 联想 | 显示 |
| 停车态 + 离线 | 键盘 | 本地缓存联想 | 显示 |
| 行车态 + 在线 | 仅语音（键盘禁用） | 语音结果直接提交 | 不显示 |
| 行车态 + 离线 | 仅语音（可能降级） | 无 | 不显示 |

**语音搜索流程**：

```
点击麦克风
  ├─ Web Speech API 可用 → 开始识别
  │    ├─ 识别中 → 实时显示文字
  │    ├─ 识别成功 → 自动提交搜索
  │    └─ 识别失败 → Toast「没听清，请重试」
  └─ Web Speech API 不可用
       ├─ JS Bridge 可用 → 走车厂语音
       └─ 均不可用 → 隐藏语音按钮，行车态下搜索入口整体隐藏
```

### IP-07 通知推送 Notification Push

**触发场景**：订单状态变更、自提取货码到达、支付结果

**统一规则**：

| 条件 | 展示方式 | 时机 |
|---|---|---|
| 停车态 | 顶栏铃铛角标 + 通知中心列表 + Toast | 实时 |
| 行车态 | 仅 Toast（紧急类：送达/即将到达） | 仅紧急类 |
| 离线 | 本地队列，上线后补推 | 恢复后 |
| 未登录 | 不推送 | — |

**通知分级**：

| 级别 | 事件 | 行车态 |
|---|---|---|
| 紧急 | 订单送达、即将到自提点、支付成功/失败 | Toast 显示 |
| 一般 | 备货中、已发货、退款进度 | 不显示，停车后补 |
| 低 | 营销活动、优惠券到期 | 不显示 |

### IP-08 错误恢复 Error Recovery

**统一策略**：

| 错误类型 | 停车态 | 行车态 |
|---|---|---|
| 网络断开 | 顶部红条「已离线，部分功能不可用」+ 本地缓存继续 | 静默降级到本地数据，不弹提示 |
| API 5xx | Toast + 重试按钮 | 静默重试 1 次，失败则使用缓存 |
| API 4xx 业务错误 | 按错误码展示具体文案 + 可操作按钮 | 静默，仅影响当前操作 |
| 购物车同步失败 | Toast「同步失败，稍后重试」+ 角标提示 | 静默，后台继续重试 |
| 订单提交失败 | 弹窗提示具体原因 + 操作引导 | Toast 原因 + "停车后可修改" |
| 支付超时 | 提示刷新二维码 + 重试 | 行车态不走扫码路径 |
| 登录态过期 | 弹登录弹层 | 内嵌二维码（P-13 模式） |

**重试策略**：

- 读操作：指数退避（1s → 2s → 4s），最多 3 次
- 写操作（幂等）：同上，带 Idempotency-Key
- 写操作（非幂等，如下单）：不自动重试，用户手动触发

### IP-09 行车态切换 Driving Mode Transition

**触发**：DrivingContext 状态变更

**进入行车态**（parked → driving）：

```
1. HTML 挂载 data-mode="driving"
2. CSS variables 自动切换（字号升档 / 动效归零 / 间距增大）
3. 当前页面执行 onEnterDriving()：
   ├─ P-01 首页 → 隐藏 B-101/B-103/B-104，保留 B-102(4项)+B-105
   ├─ P-04 商品详情 → 合并双 CTA 为"一键购买"，锁定 SKU/数量
   ├─ P-05 购物车 → 整页不可达，自动跳 /driving
   ├─ P-06 结算 → 锁定地址/支付/备注
   ├─ P-07 支付 → 如为扫码则提示切换免密
   ├─ P-08 订单 → 仅显示最近 5 单 + "再买一次"+"查看物流"
   ├─ P-10 个人中心 → 简化为订单入口 + 退出
   └─ P-13 行车态首页 → 已是行车态，无变化
4. 全局渲染 DrivingBanner（sticky 橙色横幅）
5. 转场动效 0ms（瞬切）
6. 埋点 trackEvent('driving_mode_change', { from: 'parked', to: 'driving', trigger: 'speed_threshold' })
```

**退出行车态**（driving → parked）：

```
1. HTML 挂载 data-mode="parked"
2. CSS variables 恢复
3. 移除 DrivingBanner
4. 当前页面执行 onExitDriving()（反向恢复）
5. 转场动效 240ms（平滑恢复）
6. 埋点 trackEvent('driving_mode_change', { from: 'driving', to: 'parked', trigger: 'speed_zero_3s' })
7. 如用户之前在中断页 → 弹 Toast「已恢复完整购物」+ 可跳回原页面
```

## 二、多维状态矩阵 State Matrix

> 每个页面需要回答：在 (行车/停车) × (在线/离线) × (已登/未登) 的 8 种组合下，用户看到什么、能做什么。

### 状态矩阵图例

- ✅ 完整可用
- ⚠️ 降级可用（表格内注明降级内容）
- 🚫 不可用（显示引导文案）
- 🔒 锁定（只读/默认值，不可修改）

## 三、决策树 Decision Trees

> 开发者碰到条件组合时按树走，不需要猜。

### DT-01「再买一次/一键购买」路径选择

```
用户点击「再买一次」或「一键购买」
│
├─ [未登录]
│   ├─ 行车态 → 内嵌二维码（P-13）/ 登录 Drawer（其它页）
│   └─ 停车态 → 弹登录弹层
│
├─ [已登录]
│   ├─ 行车态
│   │   ├─ 有默认地址 + 有免密支付
│   │   │   └─ POST /api/v1/orders/quick { skuId }
│   │   │       ├─ 成功 → Toast「已下单·订单号#xxx」→ 停留本页
│   │   │       ├─ STOCK_CHANGED → Toast「库存变化」→ 刷新卡片
│   │   │       └─ 失败 → Toast 错误信息 + 静默重试（幂等）
│   │   ├─ 有默认地址 + 无免密支付
│   │   │   └─ Toast「请停车后完成支付设置」
│   │   └─ 无默认地址
│   │       └─ Toast「请停车后设置收货地址」
│   │
│   └─ 停车态
│       └─ POST /api/v1/cart/quick-add { skuId, qty: 1 }
│           ├─ 在线 + 成功 → Toast「已加入购物车」→ 停留本页
│           ├─ 在线 + 失败 → Toast 错误信息
│           └─ 离线 → 写本地 + 标记 pending → Toast「已加入，联网后同步」
```

### DT-02 支付方式路径选择

```
用户到达支付环节（P-06 提交后 / 一键购买后）
│
├─ 行车态
│   ├─ 已绑定免密 → 自动走免密路径
│   │   └─ POST /api/v1/payments/:sessionId/confirm
│   │       ├─ 成功 → 绿对勾 → 跳订单详情
│   │       └─ 失败 → Toast「支付失败」+ 重试按钮
│   └─ 未绑定免密 → 显示「请停车后完成支付设置」
│       └─ 停车后 → 自动刷新为可操作状态
│
└─ 停车态
    ├─ 用户选了免密 → 同上免密路径
    └─ 用户选了扫码
        ├─ POST /api/v1/payments → 返回 sessionId + qrCode
        ├─ 显示二维码 + 倒计时 5min
        ├─ SSE 轮询支付状态
        │   ├─ CONFIRMED → 绿对勾 → 跳订单详情
        │   ├─ EXPIRED → 「二维码已失效」+ 刷新按钮
        │   └─ FAILED → 红色提示 + 重试
        └─ 用户点「换支付方式」→ 回到选择
```

### DT-03 购物车→结算路径选择

```
用户点击「去结算」（P-05 购物车底部）
│
├─ [未登录] → 弹登录弹层 → 登录后继续
│
├─ [已登录]
│   ├─ 无选中商品 → 按钮禁用（不可能到达此分支）
│   ├─ 有失效商品 → 弹窗「以下商品已失效，是否移除？」→ 移除后继续
│   └─ 正常
│       └─ POST /api/v1/orders/draft { itemIds: [...] }
│           ├─ 成功 → 跳 /checkout?draftOrderId=xxx
│           ├─ STOCK_CHANGED → 弹窗「库存变化，请确认新清单」→ 重新拉草稿
│           ├─ PRICE_CHANGED → 弹窗「价格已更新」→ 重新拉草稿
│           └─ NETWORK_ERROR → Toast「网络异常，请重试」
```

### DT-04 行车态页面跳转决策

```
DrivingContext 触发 parked → driving
│
├─ 当前在 P-05 购物车 → 强制跳 /driving
├─ 当前在 P-06 结算
│   ├─ 尚未提交 → 强制跳 /driving
│   └─ 已提交（等待响应） → 等响应完成后跳 /pay 或 /driving
├─ 当前在 P-07 支付
│   ├─ 免密进行中 → 等结果，成功后跳订单详情，失败跳 /driving
│   └─ 扫码进行中 → 提示「请停车后扫码支付」→ 保留页面但禁交互
├─ 当前在 P-12 登录
│   ├─ 扫码进行中 → 保留页面（扫码不需要驾驶员操作）
│   └─ 手机号输入中 → 切换为扫码模式
└─ 其它页面 → 执行 onEnterDriving() 降级，不跳转

DrivingContext 触发 driving → parked
│
├─ 之前在 P-05 购物车 → Toast「已恢复完整购物」+ 弹「返回购物车？」
├─ 之前在 P-06 结算（未提交）→ Toast + 弹「继续结算？」
└─ 其它 → 就地恢复，不弹跳转
```

### DT-05 网络状态变化决策

```
网络状态：online → offline
│
├─ 当前在提交操作中（下单/支付）
│   └─ 操作排队到本地，Toast「已离线，操作将在恢复后执行」
│
├─ 当前在浏览
│   └─ 顶部红条「已离线，部分功能不可用」+ 继续使用缓存数据
│
└─ 当前在登录
    └─ 二维码失效提示 + 手机号登录禁用

网络状态：offline → online
│
├─ 有 pending 购物车操作 → IP-05 批量同步
├─ 有 pending 订单提交 → 自动重试
├─ 有 pending 支付 → 提示用户确认重试（非幂等）
└─ 隐藏顶部红条 + Toast「网络已恢复」
```

### DT-06 订单操作决策

```
用户在订单页执行操作
│
├─ 「去支付」
│   ├─ 订单状态 = PENDING_PAYMENT
│   │   └─ 跳 /pay/:orderId（走 DT-02 支付路径）
│   └─ 其它状态 → 按钮不显示（不可能到达）
│
├─ 「取消订单」
│   ├─ 订单状态 ∈ { PENDING_PAYMENT, PAID }
│   │   ├─ PAID → 弹窗确认「已付款订单取消后退款至原支付方式」→ 确认后调 API
│   │   └─ PENDING_PAYMENT → 弹窗确认 → PATCH /api/v1/orders/:oid/cancel
│   └─ 其它状态 → 按钮不显示
│
├─ 「确认收货」
│   ├─ 订单状态 = SHIPPING
│   │   └─ PATCH /api/v1/orders/:oid/confirm-receipt → 成功 → 状态变 COMPLETED
│   └─ 其它状态 → 按钮不显示
│
├─ 「再买一次」
│   └─ 走 IP-01 再买一次模式
│
└─ 「申请售后」
    ├─ 订单状态 ∈ { COMPLETED, SHIPPING }
    │   └─ 跳售后页面（Demo 阶段占位 UI + mock）
    └─ 其它状态 → 按钮不显示
```
