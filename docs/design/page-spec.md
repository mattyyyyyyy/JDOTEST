## 🎨 设计调性 Design Tonality

| 关键词 | 设计语汇 |
|---|---|
| **液态玻璃 Glassmorphism** | 面板用半透明 + `backdrop-filter: blur(24px)`；卡片边缘极细高光（1px inner border with low alpha）；微噪点叠层避免"塑料感" |
| **高斯模糊 Backdrop Blur** | 弹层 / Drawer / 全局顶导 / Tab Bar 都使用 backdrop blur；模糊半径 16–32px |
| **近黑 Near-Black** | 主背景 `#0a0b0e`（非纯黑）；面板背景 `rgba(20,22,26,0.6)` + blur；强调色用车机蓝 `#3b82f6` 或电青 `#06b6d4` |
| **高对比字体** | 主文 `#f1f5f9` over near-black；正文对比度 ≥ 14:1（远超 WCAG AAA）；副文 `#94a3b8`；禁用态 `#475569` |
| **大点击区** | 所有可点击元素 `min-height: 88px`；CTA 按钮高度 96–112px；图标按钮 88×88；卡片间距 ≥ 16px |
| **演示专业感** | 微动效（200ms ease-out）；点击反馈用光晕扩散而非简单 opacity；列表项 hover 用 1px 高光带；禁止过度装饰（不要彩虹渐变 / 不要 3D / 不要 emoji 主视觉） |

## 🚧 全局通用约束

适用所有页面，不在每页重复：

- **栅格**：12 列，横屏断点 1280 / 1600 / 1920 / 2560，gap 24px
- **安全区**：上下左右各 32px 内边距，避开车机系统状态栏
- **行车态规则**（PRD §F）：
  - 隐藏所有非主操作（视频、推荐位、复杂表单）
  - 仅保留默认地址 + 默认支付 + 单击大按钮
  - 禁用任何键盘输入
  - 字号上浮 2 档（20→24, 28→40）
  - 单页操作 ≤ 1 步
- **加载态**：骨架屏（液态玻璃质感）+ 240ms 过渡
- **错误态**：浅红 toast + 重试按钮（不阻塞页面）
- **断网降级**：购物车/草稿订单本地缓存，恢复后自动同步

## 📄 页面详细规格

### 1. 首页 `/`

**定位**：用户进入车机电商的第一眼，建立"车机也能优雅购物"的印象。

**区块组成**（从上到下）：

1. **TopBar**：Logo + 搜索框（液态玻璃 pill）+ 用户头像
2. **Hero 横屏 Banner**：1 张大图 + 标题 + CTA（满屏宽，1920×360）
3. **快捷分类**：横向滑动的 8 个分类圆角图标卡（每卡 200×200）
4. **再买一次**：用户历史下单的商品，3-4 个卡片横排，1 键加购
5. **限时秒杀**：横向卡片列表，倒计时 + 折扣价
6. **推荐商品 Feed**：瀑布流 2 列（横屏 4 列），无限滚动

**核心交互**：
- 点击 Banner → 跳商品详情或活动页
- 点击分类 → `/category/:id`
- 点击商品 → `/product/:id`
- 点击"再买一次"→ 直接加购 + 跳购物车

**行车态差异**：
- 隐藏 Hero、限时秒杀、推荐 Feed
- 只保留"再买一次"区，简化为 3 张大卡 + 单击直接结算

**mock 数据字段**：
```ts
{ banner: { image, title, cta, link },
  categories: [{ id, name, icon }],
  reBuyItems: [{ id, title, price, image, lastOrderedAt }],
  flashSale: [{ id, title, originalPrice, salePrice, endsAt, image }],
  feed: [{ id, title, price, image, salesCount }] }
```

**演示价值**：让车厂方一眼看到「车机电商不是把手机塞到大屏」，而是为横屏场景重设计的。

### 2. 分类 `/category` 与分类下商品 `/category/:id`

**定位**：浏览全品类入口。

**区块组成**（`/category`）：
- 左侧：一级分类竖排列表（每项 280×96）
- 右侧：当前一级下二级分类网格（每项 240×160 卡片，图标 + 名称）

**区块组成**（`/category/:id`）：
- TopBar 含面包屑
- 左侧：筛选 Drawer 入口（停车态展开为侧栏）
- 右侧：商品瀑布流（横屏 4 列，每列宽 360px）

**核心交互**：
- 点击一级 → 右侧切换二级
- 点击二级 → `/category/:id`
- 长按商品卡 → 加入心愿单（停车态）
- 单击商品卡 → 跳详情

**行车态差异**：
- `/category`：只显示一级分类，二级隐藏
- `/category/:id`：商品改为 3 列大卡，禁用筛选

**mock 数据字段**：
```ts
{ l1Categories: [{ id, name, icon }],
  l2Categories: { [l1Id]: [{ id, name, icon }] },
  products: [{ id, title, price, image, salesCount, tags[] }] }
```

**演示价值**：横屏分类布局，与手机端"竖屏长列表"差异化对比。

### 3. 搜索 `/search`

**定位**：精准查找。

**区块组成**：
- 顶部：液态玻璃搜索框（停车态可输入，行车态置灰 + 语音按钮）
- 中部：热门搜索词（chip 形式）+ 历史记录
- 底部：搜索结果区（同分类下商品布局）

**核心交互**：
- 输入文字 → 实时联想（300ms 防抖）
- 点击麦克风 → 调起浏览器语音 API（车机环境调 JS Bridge）
- 点击 chip → 跳搜索结果

**行车态差异**：
- 键盘输入禁用
- 仅保留语音按钮（96×96 大按钮居中）
- 历史记录隐藏

**mock 数据字段**：
```ts
{ hotKeywords: ['玻璃水', '车充', '香薰'],
  history: ['特斯拉车充'],
  results: [Product] }
```

**演示价值**：演示语音搜索能力，强调行车安全分级。

### 4. 商品详情 `/product/:id`

**定位**：决策买不买的页面，电商最关键页之一。

**区块组成**（横屏左右分栏）：

**左侧 50%**：
- 主图轮播（停车态：可滑动 5-8 张图 + 视频；行车态：固定首图）
- 缩略图条（停车态显示）

**右侧 50%**：
- 商品标题（fontSize: 32, fontWeight: 600）
- 价格区（48px 大字号红色 + 划线原价）
- 规格选择（chip 形式，每个 ≥ 96px 宽）
- 服务承诺（7 天无理由 / 假一赔三 / 闪电发货）
- 库存与到达时间
- 双 CTA 按钮：「加入购物车」+「立即购买」（两个都 ≥ 112px 高）

**底部**：详情图片长图（停车态可滚动）

**核心交互**：
- 点击规格 chip → 切换价格/库存
- 加入购物车 → 顶部 toast + 购物车角标 +1
- 立即购买 → 直接进 `/checkout` 草稿订单

**行车态差异**：
- 隐藏底部长图
- 隐藏规格选择（按默认规格购买）
- 双 CTA 合并为 1 个「立即购买」（占满底部）

**mock 数据字段**：
```ts
{ id, title, images[], video?,
  price, originalPrice, sales, stock,
  specs: [{ id, name, options[] }],
  selectedSku: { id, attrs, price, stock, eta },
  services: [{ key, label }],
  detailImages: [] }
```

**演示价值**：横屏左右分栏 vs 手机竖向滚动的差异；行车态合并 CTA 的"一键购"价值。

### 5. 购物车 `/cart`

**定位**：结算前的最后核对。

**区块组成**：
- TopBar
- 主区（横屏左右分栏）：
  - **左 65%**：购物车商品列表
    - 每项一行：勾选框（96×96）+ 商品图（160×160）+ 标题/规格 + 数量增减（−/+ 各 88×88）+ 单价 + 删除
  - **右 35%**：合计面板（sticky 液态玻璃）
    - 商品件数、商品总价、优惠、运费、应付金额
    - 大 CTA：「去结算」（高 112px）
- 底部：可能推荐（停车态显示，行车态隐藏）

**核心交互**：
- 勾选/取消勾选 → 实时更新右侧合计
- 数量 +/- → 服务端同步（弱网回退到本地）
- 滑动删除（停车态）/ 长按删除（行车态禁用）
- 去结算 → `/checkout`

**行车态差异**：
- 隐藏勾选与数量编辑
- 显示"购物车 N 件商品，共 ¥XXX"
- 单个「全部结算」大按钮

**mock 数据字段**：
```ts
{ items: [{ skuId, title, image, specText, price, quantity, selected, stock }],
  summary: { itemCount, productAmount, discount, shipping, payable } }
```

**演示价值**：右侧 sticky 合计面板的液态玻璃效果是设计重点。

### 6. 结算 `/checkout`

**定位**：填写收货 + 选支付 + 提交订单。

**区块组成**（横屏分栏）：

**左 60%**：
- **收货方式**：tab 切换「自提点 / 配送到家 / 送到车上」
- **地址卡片**：选中态高光，可点击切换
- **商品清单折叠区**（默认折叠）
- **订单备注**：单行输入

**右 40%**（sticky）：
- 价格明细（与购物车一致）
- **支付方式**：单选卡片（微信扫码 / 支付宝扫码 / 免密支付）
- 协议勾选
- 大 CTA：「提交订单」（高 120px）

**核心交互**：
- 切换地址 → 重新计算运费 + ETA
- 切换支付 → 仅 UI 切换，提交后跳 `/payment/:orderId`
- 提交 → 后端创建订单 → 跳支付

**行车态差异**：
- 锁定为「默认地址 + 默认支付方式」
- 隐藏备注、商品清单
- 仅一个大「确认下单 ¥XXX」按钮

**mock 数据字段**：
```ts
{ deliveryKind, addresses[], defaultAddressId,
  items[], summary,
  paymentMethods: [{ id, name, icon }] }
```

**演示价值**：「送到车上」这种车机独有的履约方式，是核心差异化卖点。

### 7. 支付 `/payment/:orderId`

**定位**：等待支付完成。

**区块组成**：
- 中央液态玻璃大卡（800×600）
  - 金额（72px 大字号）
  - **二维码**（400×400，居中）
  - 提示文案："请用手机扫码完成支付"
  - 倒计时（5 分钟）
  - 已选支付方式 + 切换按钮
- 底部：取消支付（弱化按钮）

**核心交互**：
- 每 2 秒轮询订单状态
- 状态变 paid → 跳 `/orders/:id` 成功页
- 超时 → 显示重试 + 联系客服

**行车态差异**：
- 二维码放大到 600×600
- 文案改为"请副驾扫码"
- 取消按钮隐藏

**mock 数据字段**：
```ts
{ orderId, amount, qrCode (data URL), method, expiresAt, status }
```

**演示价值**：「车机扫码 + 手机支付」是规避车内输密码风险的标准方案，演示重点。

### 8. 订单列表 `/orders` 与详情 `/orders/:id`

**定位**：跟踪与售后入口。

**区块组成（列表）**：
- TopBar + tab 切换（全部 / 待付款 / 待发货 / 待收货 / 已完成）
- 订单卡片列表（每张卡液态玻璃，含商品缩略图、状态徽章、价格、操作按钮）

**区块组成（详情）**：
- 顶部状态条（图标 + 状态 + 描述，60px 高）
- 物流/自提信息卡片
  - 自提：地址 + 大字号取货码（96px）
  - 配送：物流轨迹时间线
- 商品清单
- 价格明细
- 底部操作按钮（再次购买、申请售后、删除订单）

**核心交互**：
- 点击订单卡 → `/orders/:id`
- 点击「再次购买」→ 直接加购 + 跳购物车
- 详情页轨迹自动刷新

**行车态差异**：
- 列表只显示最近 3 单
- 详情隐藏轨迹细节，只显示状态 + ETA

**mock 数据字段**：
```ts
{ orders: [{ id, status, items[], totalAmount, createdAt, fulfillment }],
  detail: { ...order, tracking: [{ ts, status, location, desc }], pickupCode? } }
```

**演示价值**：自提码大字号设计 + 物流轨迹时间线，体现「车主取货」场景的优化。

### 9. 我的 `/me`

**定位**：账号管理入口。

**区块组成**：
- 顶部头像 + 昵称 + 手机号（脱敏）
- 数据栏：订单数 / 收藏 / 优惠券 / 积分（4 个卡片横排）
- 功能列表：地址簿 / 客服 / 设置 / 关于
- 底部：退出登录按钮（红色弱化）

**行车态差异**：
- 简化为头像 + 1 个"我的订单"大按钮 + 退出登录

**mock 数据字段**：
```ts
{ user: { avatar, name, phone (masked) },
  stats: { orderCount, favCount, couponCount, points },
  menus: [{ key, label, icon }] }
```

**演示价值**：低密度，主要体现深色 + 液态玻璃下的"信息层次"。

### 10. 登录 `/login`

**定位**：未登录时的统一入口。

**区块组成**（居中卡片，800×600 液态玻璃）：

**主区**：
- Logo + Slogan
- **二维码（500×500）**：核心入口（车机扫码登录）
  - 二维码下方："请用手机/JDO App 扫码登录"
- 倒计时 + 刷新按钮
- 状态指示（等待扫码 / 已扫码待确认 / 登录成功）

**底部**：
- 弱化文字链："手机号验证码登录" → 展开输入区（停车态）
- 用户协议勾选

**行车态差异**：
- 隐藏手机号输入入口
- 二维码放大到 600×600
- 文案改为"请副驾或乘客扫码"

**核心交互**：
- 进入页面自动请求 sessionId + qrCode
- 每 2 秒轮询 qr-status
- confirmed → 跳回来源页或 `/`

**mock 数据字段**：
```ts
{ sessionId, qrCode (data URL), expiresAt, status }
```

**演示价值**：完整演示"车机不输密码不输手机号"的安全登录方案，对车厂方说服力强。

## 🚀 首批要落代码的页面（建议）

按演示价值 × 实现成本，**第一批做 5 页**就能跑通主链路 Demo：

1. **首页 `/`** — 第一印象 + 入口
2. **商品详情 `/product/:id`** — 横屏分栏的视觉高潮
3. **购物车 `/cart`** — 液态玻璃 sticky 合计
4. **登录 `/login`** — 车机扫码场景的标志性页面
5. **支付 `/payment/:orderId`** — 二维码支付演示

剩余 5 页（分类 / 搜索 / 结算 / 订单 / 我的）放第二批。
